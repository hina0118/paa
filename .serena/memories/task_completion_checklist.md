# タスク完了時のチェックリスト

## コード変更後の必須タスク

### 1. TypeScriptコンパイルチェック
```bash
tsc
```
- 型エラーがないことを確認
- コンパイルが成功することを確認

### 2. ビルド確認
```bash
npm run build
```
- ビルドプロセスが成功することを確認
- ビルド成果物が正しく生成されることを確認

### 3. 開発サーバーでの動作確認
```bash
npm run dev
```
- アプリケーションが正常に起動することを確認
- 変更した機能が期待通りに動作することを確認
- UIが正しく表示されることを確認

## オプション確認項目

### Rustコード変更時
```bash
cd src-tauri
cargo check
cargo build
```

### 新しいコンポーネント追加時
- コンポーネントが正しくエクスポートされているか確認
- インポートパスが正しいか確認
- TypeScript型定義が適切か確認

### スタイリング変更時
- Tailwind CSSのクラスが正しく適用されているか確認
- レスポンシブデザインが維持されているか確認
- ダークモード対応 (必要な場合) が機能しているか確認

### 依存関係追加時
```bash
npm install
```
- package.jsonが正しく更新されているか確認
- 依存関係の競合がないか確認

## Git関連

### コミット前
```bash
git status
git diff
```
- 意図しないファイルがステージングされていないか確認
- 変更内容が適切か確認

### コミット
```bash
git add .
git commit -m "適切なコミットメッセージ"
```
- わかりやすく簡潔なコミットメッセージを記述

## トラブルシューティング

### ビルドエラー時
1. `npm install` で依存関係を再インストール
2. `node_modules` を削除して再インストール
3. TypeScriptエラーを確認・修正
4. Vite設定を確認

### 開発サーバー起動失敗時
1. ポート1420が使用中でないか確認
2. Tauriプロセスが残っていないか確認
3. `src-tauri/target` をクリーンアップ

### 型エラー時
1. `tsc --noEmit` で詳細なエラー情報を取得
2. 型定義ファイル (`*.d.ts`) を確認
3. `tsconfig.json` の設定を確認
